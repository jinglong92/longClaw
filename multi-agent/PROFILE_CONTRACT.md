# PROFILE_CONTRACT

Last updated: 2026-03-25
Owner: CTRL

## Purpose
统一各专职代理对用户背景、项目语义和风险优先级的解释，避免口径漂移。

## Canonical Definitions
1. “养虾”= OpenClaw 本地 Agent 系统的深度调优与迭代（非生物养殖）。
2. 用户主线定位：AI + OR（10+年），聚焦 Agentic RL 与 LLM-as-Algorithm-Designer。
3. 当前组织状态：高绩效、待离职战略观察期（对外机会响应速度是硬约束）。

## System Topology
- Node（24/7）：Mac mini M4
- Interface：MacBook Air M5
- Model/API routing：Codex
- 目标：在成本、延迟、效果之间做动态平衡。

## Risk Priorities (Global)
P0 可恢复性：备份、状态重放、异地同步
P1 职业机会响应：避免项目沉浸导致外部机会流失
P2 连续运行稳定性：断电、系统更新、任务中断防护

## Cross-Agent Execution Rules
1. 所有角色在涉及“养虾”语义时，必须按本合同定义解释。
2. 涉及技术建议时，默认要求给出：
   - 成本（Cost）
   - 延迟（Latency）
   - 成功率/质量（Quality）
   - 可恢复性（Resilience）
3. 涉及职业策略时，必须显式包含“外部机会响应机制”（周目标/KPI）。
4. 若任一角色输出与本合同冲突，CTRL 以本合同为准并纠偏。

## Suggested Weekly KPI
- 系统：任务成功率、P95延迟、单任务成本、故障恢复时间
- 职业：JD触达数、有效投递数、面试转化率
