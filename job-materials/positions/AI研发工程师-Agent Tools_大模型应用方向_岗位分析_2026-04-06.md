### 职责摘要
1. 基于大模型（LLM）设计与研发 Agent 系统，构建面向复杂任务的智能体能力。
2. 设计并实现通用 Agent 工具体系，包括：
   - Tool/Skill 抽象与管理机制
   - 多工具协同调用（Tool Orchestration）
   - Agent 执行框架（Planning / Memory / Execution 等）
3. 参与构建面向学术与工程场景的智能系统，包括自动化实验评估、智能代码生成执行、知识检索分析系统。
4. 持续优化复杂任务下的大模型效果与稳定性，重点包括 Prompt 设计优化、Tool Calling 能力增强、Evaluation Pipeline 建设。
5. 跟踪大模型与 Agent 前沿并推动业务落地。

### 任职要求补充（来自新增截图）
1. 学历背景：硕士及以上，计算机/AI/软件工程相关（优秀本科可放宽）。
2. 工程基础：扎实编程能力，熟练 Python（或至少一门主流语言），强调代码规范与系统设计能力。
3. LLM 基础：熟悉主流大模型（如 GPT/GLM/Claude/开源模型）的使用与原理。
4. 应用经验：要求有完整 LLM 应用开发与落地经验。
5. Agent 经验：理解或实践 Agent 技术，熟悉任务分解/工具调用/记忆管理链路。
6. 框架经验：熟悉常见 Agent/LLM 应用框架者优先（LangChain、LlamaIndex、AutoGPT 等）。

## 与你当前主线匹配度（快速）
- 匹配度：**高（A-）**
- 匹配原因：
  - 与你“LLM/Agent + 系统化编排 + 工程化落地”的主线高度一致。
  - 关键词与当前 OpenClaw/多代理系统调优经验直接重叠，故事可讲性强。
- 潜在短板：
  - 年限标注 3-5 年，可能存在“资深度与岗位层级”错配。
  - 标题是研发工程师，若团队偏执行交付，研究/策略空间可能受限。

## 风险审计（Risk Audit）
1. **层级错配风险（高概率/中等损失）**：薪资上限和年限画像可能更偏中高级工程师，不一定匹配你希望的影响力范围。
2. **职责边界风险（中概率/中高损失）**：若岗位被定义为“工具链实现+业务接入”，算法主导权可能不足。
3. **预期管理风险（中概率/中等损失）**：若面试前不校准“技术决策权/KPI性质”，容易后期认知偏差。

## 建议动作（下一步）
1. 先沟通 3 个校准问题：
   - 这个岗位的真实级别、汇报对象、是否带人？
   - 核心 KPI 是模型效果、业务收益，还是交付速度？
   - 是否支持“LLM+求解器/启发式”这类算法路线探索，而不只是工程接入？
2. 若对方回答偏“执行型工程岗”，将其定位为备选，不作为第一优先。
3. 若确认可做架构与算法路线主导，再投入定制化简历与项目故事（突出 Agentic RL + OR 融合能力）。
