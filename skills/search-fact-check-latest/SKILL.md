# Fact Check Latest

核查用户询问的最新信息，多源交叉验证，给出有时效标注的可信结论。

## 触发条件
- 用户问"现在 X 多少钱" / "最新的 X 是什么" / "X 有没有更新"
- 用户说"帮我查一下" / "确认一下这个信息"
- 需要核查的信息涉及价格/政策/技术版本/资讯

## 检索流程

### Step 1：Query 改写
将用户问题改写为 2-3 个搜索变体：
- 精确版：直接搜索目标信息
- 宽泛版：搜索相关背景
- 时间限定版：加上"2026"或"最新"限定

### Step 2：多源检索
按优先级顺序：
1. 官方源（官网/官方文档/官方 API）
2. 权威聚合源（行业报告/主流媒体）
3. 社区源（GitHub/Stack Overflow/Reddit）

### Step 3：交叉验证
- ≥2 个独立来源一致 → 确定信息（F）
- 只有 1 个来源 → 推断信息（I），标注不确定性
- 来源冲突 → 列出分歧，说明可能原因

### Step 4：结果输出
```
查询：<用户问题>
获取时间：<YYYY-MM-DD HH:MM>

结论：
[F] <确定信息>（来源：<URL>）
[I] <推断信息>（来源：<URL>，置信度：X%）

来源列表：
1. <URL> - <简要说明>
2. <URL> - <简要说明>

时效说明：<该信息的有效期估计>
注意事项：<如有价格波动/政策变化风险>
```

## 主动排障
如果第一个来源失败，自动按以下顺序尝试备用路径：
1. 官方 API / RSS
2. Wayback Machine（web.archive.org）
3. 其他搜索引擎
不直接向用户求助，除非所有路径都失败
