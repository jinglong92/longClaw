# JD Analysis

当用户发送岗位 JD 文本、截图或链接时，自动执行以下分析流程。

## 触发条件
- 用户发送 JD / 岗位描述 / 职位要求
- 用户说"帮我分析这个岗位" / "这个 JD 怎么样" / "匹配度如何"

## 分析流程

### Step 1：岗位解码（3 分钟）
提取：
- 硬技能要求（必须 vs 加分）
- 软技能信号（团队/独立/跨部门）
- 隐含要求（从用词/行业背景推断）
- 薪资区间估计（如有）

### Step 2：匹配度评级
| 评级 | 含义 | 建议 |
|------|------|------|
| A | 高度匹配（≥80%要求覆盖） | 主投，本周内投递 |
| B+ | 较好匹配（60-80%） | 次投，优化简历后投递 |
| B | 一般匹配（40-60%） | 兜底，视情况投递 |
| C | 低匹配（<40%） | 暂不投，除非战略性目标 |

### Step 3：简历叙事建议
针对该 JD 的核心要求，给出：
- 需要突出的 1-3 个项目经历
- 需要补充的量化指标
- 建议删除或弱化的内容

### Step 4：投递行动
- 投递平台建议
- 内推可能性分析
- 本周行动清单（P0/P1/P2）

## 输出格式
```
岗位：<公司名> - <职位名>
评级：<A/B+/B/C>
匹配度：<X%>

核心要求：
- [必须] ...
- [加分] ...

简历叙事建议：
- 突出：...
- 补充：...

本周行动：
- P0: ...
- P1: ...

[置信度: X.XX] [依据: JD文本分析]
```
