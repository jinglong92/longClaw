执行完整性 · Deny > Ask > Allow
三层授权模型，6 条 Immutable Rules。doing: 仅在同轮已发起工具调用时合法，done: 必须有执行证据（diff / stdout / commit hash）。

设计借鉴
逆向学习 · 本地化改造 · 非盲目迭代

longClaw 的核心设计不是凭个人经验盲目迭代，而是对 Claude Code 内部 Harness 架构和 Hermes Agent 框架进行逆向分析后，提取可迁移的设计模式，再结合本地场景做改造落地。以下是三个主要借鉴来源及其具体映射。

①
借鉴 Claude Code — 权限模型 · Hook 工程 · 压缩设计
sourcemap 逆向分析 → 三维度落地
Claude Code 原设计
Deny > Ask > Allow 三层权限
Deny 规则优先于所有 hook，任何 prompt 指令不得覆盖，解决规则冲突时 LLM 自由裁量的漏洞。
↳ longClaw 落地
写入 6 条 Immutable Rules（无合成证据、禁 force-push main、SOUL 对所有专职生效等），任何 skill 或用户指令不得覆盖。
Claude Code 原设计
Hook Config Snapshots
PostCompact 后自动重注入关键配置，FileChanged 监听配置变更，PreToolUse 在执行层拦截危险命令。
↳ longClaw 落地
PostCompact 重注入 CTRL_PROTOCOLS.md + DEV_LOG.md；PreToolUse 把 rm 改写为 trash，不靠 LLM 自觉。
Claude Code 原设计
Tool Result Budgeting
每次工具输出做 budget 限制，超出截断，防止单条超长输出污染后续检索 context。
↳ longClaw 落地
新增 Layer 1 Trim：单次工具输出 > 500 字符立即截断，静默执行，不计入 compression_count。
②
借鉴 Hermes Agent — Skill 依赖声明 · 自进化提议系统
Anthropic 内部多 agent 框架 → 本地化改造
Hermes 原设计
requires 依赖声明
Agent 执行前检查前置条件，不满足时明确失败而非假装执行。把运行时失败提前到启动前，给用户明确的失败信号。
↳ longClaw 落地
每个 SKILL.md frontmatter 新增 requires 字段，CTRL 命中触发条件后先验证依赖，不满足返回 blocked: missing_tool，不空转。15 个 skill 全量迁移，分四类：无依赖 / 需 web_fetch / 需 file_read / 需 file_write。
Hermes 原设计
Proactive Skill Creation 自进化
Agent 能自主发现可复用模式并生成新 skill，减少用户重复描述同一流程的认知负担。
↳ longClaw 落地（加安全边界）
保留"主动提议"理念，但需用户确认后才固化为 SKILL.md。触发优先级：复杂流程完成后（≥3步+可复用）当轮末尾主动提议；重复模式检测阈值从 ≥3 次降至 ≥2 次。
③
自研迭代 — 被问题逼出来的工程经验
非借鉴，是实践中发现的规律
📐
单体文件拆分经验法则
684 行 MULTI_AGENTS.md 把专职定义、路由规则、Skill 协议、DEV LOG 格式全混在一起，LLM 处理时后半段规则频繁遗忘。按关注点分离拆成三个职责单一文件（175 / 177 / 149 行），每个文件头部声明"本文件管什么、不管什么"。
经验法则：单配置文件 > ~200 行警惕，> 400 行必然出现遗忘/幻觉
💓
Heartbeat 主动巡检
原来只能等用户问"我有什么待办"，Heartbeat 把被动等待变成主动推送。cron 08:30/18:00 触发巡检，P0/P1 事项写入 heartbeat-state.json，SessionStart hook 在用户开口时自动呈现。P2 系统健康检查还会自动修复：发现 inbox/ 有新文件自动运行处理器，发现索引 >90 天自动重建。
