Heartbeat Agent 主动巡检 — 从被动等待到 SessionStart 感知
架构
cron 08:30/18:00 → heartbeat-agent 巡检 → 写 heartbeat-state.json → SessionStart hook 读取 → 用户开口时呈现
三级优先级
P0：今日/明日截止事项
P1：近7天持续跟进
P2：系统健康（索引老化、inbox 新文件）
自动修复
P2 检测到 inbox/ 有新文件 → 自动运行 inbox_processor.py
索引 >90天 → 自动重建
Coding Agent repo-explorer + code-agent — 5步编排闭环
repo-explorer（只读子代理）
6步探索：目标理解→结构扫描→关键词定位→深度读取→依赖追踪→输出结构化文件地图
code-agent skill（编排层）
spawn repo-explorer → 制定计划（用户确认）→ 执行修改 → 运行测试（最多重试2次）→ 交付报告
失败换路策略
第1次失败 → 检查测试本身
第2次失败 → 缩小修改范围
第3次失败 → 停止，报告 blocked
检索修复 四个模块的设计缺陷修复 — 会话状态 · 压缩 · 检索 · 记忆
检索扩展条件
差值 top1-top2 < 0.05（过敏感，频繁触发跨域）
→ 改为绝对值 top1 < 0.3，防止低分噪声误触
记忆老化检测
importance < 0.4 且 >90天 → 输出 [stale] 列表 → 自动重建索引
压缩优先级
