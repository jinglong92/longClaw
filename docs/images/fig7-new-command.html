/new 命令 — 上下文清空流程

UserPromptSubmit Hook 检测 → openclaw gateway restart → 干净新会话
1
用户发送 /new
微信 / WhatsApp / Telegram 渠道均可触发
字符串精确匹配，不需要语义推断
2
UserPromptSubmit Hook 触发
OpenClaw 在发送给模型前拦截
检测到 /new → 执行钩子命令
3
openclaw gateway restart
重启 Gateway 进程 → 真正清空 context window
等同于原生 /new — 旧对话彻底丢弃
4
新 Session 启动
Bootstrap Files 重新注入（AGENTS / SOUL / USER / IDENTITY / TOOLS）
SessionStart Hook → 检查 heartbeat-state.json → 呈现待办
5
干净会话就绪
零历史上下文 → 无旧对话干扰 → 从 Bootstrap 开始
记忆召回仍可用（memory_search 从 JSONL 索引读取）
⚠️ 关键：先归档再 /new
/new 不可逆，旧 context 直接丢弃
正确顺序：先说"归档" → Layer 4 Archive → 再发 /new
反向操作 = 数据丢失
⚙️ Hook 配置
.claude/settings.json 中配置：
"UserPromptSubmit": [{
  "matcher": "/new",
  "hooks": [{
    "type": "command",
    "command": "openclaw gateway restart"
  }]
}]
✅ 记忆不丢失
/new 只清空 context window（对话历史）
文件系统不受影响：
MEMORY.md · memory/*.md · JSONL 索引
新 session 仍可通过 memory_search 取回
🔄 ephemeral session
微信 bot 每条消息 = 新 session
/new 对 CLI 场景更有意义（单 session 长对话）
ephemeral 不跨 session 累积状态
