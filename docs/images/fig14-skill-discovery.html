症状：仓库里明明有 skill，聊天里却不命中。真实原因：skill 根本没进 registry，谈不上命中。

🔍 现象与证据
症状
skill 有内容、有触发条件，聊天里就是不命中
openclaw skills check
59 个 skill / 22 eligible / 0 blocked by allowlist
→ allowlist 没有拦截，问题在更前面
skills list --json
source=openclaw-workspace 只有 2 个：
multi-agent-bootstrap 和 paperbanana
关键差异
这两个恰好是 skills/ 下一层的 skill，其余全是两层分组目录
💡 根因
OpenClaw workspace skill discovery 只扫 skills/ 下一层目录，不递归扫第二层分组目录。

skills/<skill>/SKILL.md → 进 registry
skills/<group>/<skill>/SKILL.md → 隐身，不进 registry
排查路径（下次遇到 skill 不命中，按此顺序）
1
查 registry
skills list --json
看 source=workspace 有几个，和磁盘上实际数量对比
›
2
查目录层级
registry 数量 < 磁盘数量 → 检查目录是否超过一层
›
3
最小验证
拍平一个 skill 到一层，重启 session，再查 registry
›
4
查 frontmatter
进了 registry 但仍不命中 → 检查 name/description/触发条件
›
5
新开 session
改完目录后必须新开 session，旧会话沿用老的 skill 列表快照
