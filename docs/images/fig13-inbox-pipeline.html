💡 设计动机
痛点 对话记忆只记"发生了什么"，学到的技术文章、论文笔记无法进入检索范围

解法 双轨索引：memory_entries（对话） + knowledge_entries（知识库），load_all() 统一检索入口

知识条目与对话记忆共用同一套 FTS + 实体打分机制，无需维护两套检索逻辑
📄 knowledge_entry 结构
id：kb_ + md5 前8位
source_type：knowledge（区别于 memory）
domain：JOB / LEARN / ENGINEER…
entities：自动提取的实体列表
importance：0.1–1.0（frontmatter 或自动估算）
status：active / archived
