🔐
Deny > Ask > Allow
三层权限模型
借鉴 Claude Code
原来（2层）：Allow / Require auth，冲突时 LLM 自由裁量
Deny 永久禁止，优先于所有 hook，不可被指令覆盖
Ask 每次单独授权；Allow 默认允许
Immutable Rules（6条）：无合成证据 / 禁 force-push / SOUL 对所有专职生效 / DEV LOG 每轮必须输出…
防止"用话术绕过禁止操作"的越权攻击
🪝
三类 Hook
Harness 层强制执行
借鉴 Claude Code
PostCompact：原生压缩后自动重注入 CTRL_PROTOCOLS.md + DEV_LOG.md，防止规则消失
FileChanged：监听配置文件变更（AGENTS.md / MULTI_AGENTS.md），自动提示重读
PreToolUse：rm 命令自动改写为 trash，Harness 层执行，不靠 LLM 自觉
Hook Config Snapshots：启动时冻结配置，防止运行时注入
📦
四层压缩协作
Tool Result Budgeting
借鉴 Claude Code
Layer 1 Trim（借鉴 Tool Result Budgeting）：单次输出 >500字符立即截断，防止单条污染 context
Layer 2 Summarize：round > 20 触发，生成摘要块替换中间历史
Layer 3 Compact：OpenClaw 原生压缩，接近上限时触发
Layer 4 Archive：话题切换时提炼 key_conclusions 写入 MEMORY.md，跨 session 持久化
PostToolUse 注入
每次工具调用完成后，结果摘要注入 DEV LOG 🛠️ 字段，最低合格线 8字段 → 9字段

Fork Agent 隔离
search-agent / memory-agent 继承父 context，独立执行，权限最小化（只读/只查）

核心收获
