OpenClaw 上下文组装流程

1
用户发消息
OpenClaw Gateway 收到请求 → 开始上下文组装
↓
2
Session 恢复
查 sessions.json → 找到 sessionKey → 复原 JSONL transcript
没有 → 新建 session
→
↓
3
注入 Bootstrap Files（每次都注入）
AGENTS.md → 操作规程 SOUL.md → 人格/语气 USER.md → 你是谁 IDENTITY.md → 它叫什么 TOOLS.md → 工具备注
→
↓
4
Agent 按 AGENTS.md 规则执行
memory_search 语义检索  |  memory_get 读今天/昨天日志
→
↓
5
组装 Context → 发给模型 → 回复
Bootstrap + Session 历史 + 检索到的记忆 + 当前用户消息
↓
6
对话中 → 记忆写入 & 自动压缩
重要事项 → 写入 memory/YYYY-MM-DD.md
快到上限 → Silent memory flush → Auto-compaction 压缩旧对话
📄 MEMORY.md
长期记忆：用户身份、偏好、关键决策
跨 session 持久保存
📄 memory/*.md
每日日志：任务进度、临时上下文
按日期分文件：YYYY-MM-DD.md
📄 sessions.json
会话索引：sessionKey → JSONL 路径
用于恢复对话历史
📄 *.jsonl transcript
完整对话记录：每条消息一行 JSON
按需加载、压缩旧轮次
Context
Bootstrap + History + Memory + User Msg
