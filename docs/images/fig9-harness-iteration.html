现象

Codex 处理后半段时遗忘前半段规则
DEV LOG 字段频繁省略或幻觉替换
根因
专职定义 / 路由 / Skill协议 / DEV LOG格式
全混在一个文件，规则相互干扰
LLM 位置偏差：前半段权重 > 后半段
解法
684行 → 3×~175行
MULTI_AGENTS.md 只管路由
CTRL_PROTOCOLS.md 只管协议
DEV_LOG.md 只管格式
借鉴 Claude Code Deny > Ask > Allow 三层权限 + Immutable Rules
原来（2层）
Allow by default
Require explicit auth
无优先级，规则冲突时 LLM 自由裁量
借鉴后（3层）
Deny → 永久禁止，任何指令不得覆盖
Ask → 每次单独授权
Allow → 默认允许
Deny 优先于所有 hook
Immutable Rules（6条）
无合成证据 / 无静默改 AGENTS.md
禁 force-push main
Deny>Ask>Allow 固定
SOUL 对所有专职生效
DEV LOG 每轮必须输出
借鉴 Claude Code PostToolUse 注入 + PostCompact 重注入
PostToolUse → DEV LOG 工具字段
每次工具调用后结果摘要注入 DEV LOG
格式：🛠️ 工具 Read(x.md) → ok
status=blocked 时说明原因
最低合格线：8字段 → 9字段
PostCompact → 协议重注入
原生压缩后 CTRL_PROTOCOLS.md
和 DEV_LOG.md 会从 context 消失
PostCompact hook 自动重注入
→ 压缩后规则不再丢失
PreToolUse → rm 拦截
rm 命令自动改写为 trash
Harness 层执行，不靠 LLM 自觉
即使 LLM 输出 rm，实际不会删除
借鉴 Hermes Skill requires 依赖声明 + Proactive 提议系统
requires 字段
SKILL.md frontmatter 声明依赖
requires: [web_fetch]
命中触发条件后先验证依赖
不满足 → blocked: missing_tool
提议系统升级
主触发：复杂流程完成后（≥3步）
重复模式阈值：≥3次 → ≥2次
优先级：任务完成后提议
> 用户明确表达 > 重复模式
