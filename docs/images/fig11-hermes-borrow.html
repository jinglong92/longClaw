Skill requires 依赖声明
借鉴 Hermes
SKILL.md frontmatter 新增 requires 字段，声明该 skill 运行所需的工具能力
CTRL 在命中触发条件后、加载 SKILL.md 全文之前，先检查 requires
不满足 → 返回 blocked: missing_tool(web_fetch)，不得假装执行或空转
Before
skill 命中后直接执行，web_fetch 不可用时 CTRL 自己搜一遍就完事，没有明确的失败信号
After
requires: [web_fetch] → 运行前检查 → 不满足时给出 blocked 原因 + 降级建议（如"提供直链或 PDF"）

Proactive Skill Creation — 自进化提议系统
借鉴 Hermes 自进化理念
Hermes 原设计
agent 自主发现可复用模式，自动生成新 skill，无需人工介入
longClaw 本地化
保留"主动提议"，但需要用户确认后才固化为 SKILL.md（安全边界）
触发优先级重排（对齐 Hermes 自进化理念）：
① 复杂流程完成后（≥3步 + 可复用）→ 当轮末尾主动提议
② 用户明确表达"每次都要..."
③ 重复模式检测（≥2次，从 ≥3次 降低）
提议格式：一句话说明 + 建议 skill 名 + 是否现在创建
