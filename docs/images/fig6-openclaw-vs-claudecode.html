上下文组装：OpenClaw vs Claude Code 设计差异

OpenClaw
显式记忆 + 多层压缩 + 语义检索
1
Gateway 收到消息
查 sessions.json → 找 sessionKey → 复原 JSONL transcript
↓
2
注入 5 个 Bootstrap Files
AGENTS.md SOUL.md USER.md IDENTITY.md TOOLS.md
↓
3
4 级 Route-Aware 语义检索
memory_search → L1 本域 → L2 域间 → L3 全局 → L4 文件系统
memory_get → 读取今天/昨天日志
↓
4
组装 Context → 调用模型
Bootstrap + 检索记忆 + Session 历史 + 用户消息
↓
5
4 层压缩协作
L1 Trim 截断 L2 Summarize 摘要 L3 Compact 原生 L4 Archive 归档
Hook: PostCompact 重注入丢失文件
↓
6
显式记忆写入
MEMORY.md 长期记忆 + memory/YYYY-MM-DD.md 每日日志
话题结束 → 写入 + 重建索引 + 告知用户
VS
Claude Code
隐式压缩 + CLAUDE.md 优先级栈 + 附件系统
1
QueryEngine 收到消息
JSONL 持久化 → 自动恢复对话
无显式 session 索引文件
↓
2
CLAUDE.md 优先级栈（6 层）
/etc 管理策略 ~/.claude 全局 项目 CLAUDE.md CLAUDE.local.md AutoMem MEMORY TeamMem
目录从 root→CWD 向上遍历，近 CWD 优先级更高
↓
3
30+ 附件类型（Per-Turn）
每轮动态组装：文件内容、IDE 集成、记忆片段、
Todo 提醒、Hook 结果、技能注入、Swarm 协调等
1 秒超时保护
↓
4
组装 Context → 调用模型
System Prompt + CLAUDE.md + Attachments + 消息历史
静态前缀可全局缓存（跨 session 复用）
↓
5
3 层压缩策略
MicroCompact 清旧输出 Session Memory 无 API API 摘要兜底
PostCompact: 50K token 重新注入预算
↓
6
隐式 AutoMem 写入
~/.claude/memory/MEMORY.md Agent 自行管理
无结构化索引、无域划分、无老化检测
核心差异
OpenClaw：显式记忆体系（结构化 MEMORY.md + daily logs + 4 级语义检索 + 4 层压缩 + 老化检测 + 索引重建），记忆可检索、可管理、可过期
Claude Code：隐式记忆体系（CLAUDE.md 优先级栈 + AutoMem + 附件系统 + 3 层压缩），记忆靠文件优先级覆盖，无结构化检索，但缓存策略精细
