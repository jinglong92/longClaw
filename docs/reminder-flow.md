# Reminder Flow

## Purpose

Provide true due-time proactive reminders, separate from silent heartbeat.

## Boundary

- `heartbeat` remains silent inspection + show-on-next-user-message.
- `reminder` handles actual due-time proactive delivery.

## Files

- `memory/reminders.json`: reminder registry
- `memory/due-reminders.json`: due reminders generated by poller
- `scripts/run_due_reminders.py`: computes due reminders and marks them sent
- `scripts/install_reminder_cron.sh`: installs Gateway cron for reminder polling

## Current delivery design

Gateway cron runs every 5 minutes and asks the main agent to:
1. run `scripts/run_due_reminders.py`
2. read `memory/due-reminders.json`
3. if any item is due, send proactive messages into the target session

## Current state machine

`pending -> due -> sent|failed`

Important: due detection must not mark an item as `sent` before actual delivery success is confirmed.

## Reminder item schema

```json
{
  "id": "reminder_2026_04_22_transfer",
  "title": "办理指标过户",
  "content": "去朝阳交通管理大队",

  "chat_id": "o9cq802fWfRMLbme2kzXdHFEwIyw@im.wechat",
  "reply_tag": "[[reply_to_current]]",
  "status": "pending"
}
```
